       pmk.jar  (Ver 1.0)

This JAR file contains three class files :                                                                                                          ReportBeanException.class : It is a general exception class .

   ReportBeanProperties.class : It consists of all the methods that a ReportViewerBean contains and,in your code development ,you are supposed to :   

a)    Create an instance of ReportBeanProperties ( Using a No Argument Constructor )                                 i.e. ,    ReportBeanProperties b = new ReportBeanProperties();             

b)    Set the necessary properties ( Please check the doc "ListOfProperties") to a report                  eg. , b.setReportName("http://servername:80/reportname.rpt");                                b.setBackground(Color.green);      

c)     i)  Call init() to View a report on the Smart Viewer                                                                               ii)  Call printReport() to Print the Report directly to a printer                                                                  iii)  Call exportReport(File f) or exportReport(int type,File f) inorder to Export the report to              .doc,.xls,.rpt,.rtf  format.                                                                                                              Please note :    Create a fresh instance of ReportBeanProperties each time a button is pressed .

 ReportBeanProperties$MyWindowAdapter.class : Simply include this class file along with other class files. It is used to destroy the frame incase you want to close the report after viewing. Don't worry incase you are not able to understand what I am saying as the SampleCode.java file will clarify your doubts further.

      Report.java 

This JAVA file is a Sample Code explaining the usage of the various properties of the bean.Please note:                                                                                                                                          While creating your own Java code , you can make your Java file in pmk.com.report          package  itself  as shown or import this package in your code as:                                               import pmk.com.report.* ;                                                                                        inorder to use these class files.                                                                                    Change the string being passed to : setReportName,setReportParameters according to          View/Export/Print a desired report.                                                                                                           Change the Property values acc. to the requirements.The List of Properties that can be used are listed in the ListOfProperties.doc .

Please note :                                                                                                                                               Unzip the contents of the jar files to the same folder as your application ,preserving the  path.ie.,say ,you try unzipping pmk.jar file.Since,I have included it in pmk.com.report  package;So,unzipping the contents of this jar file will create a sub-folder pmk\com\report where all  the class files will be available.So,Do NOT add the class files directly to your  application path.Instead,add them to <application_name>\pmk\com\report folder.                                    Incase you are using some tool like, JBuilder or JDeveloper, you can directly refer to JAR files.Please edit the User Information as required by your Report.

This document discusses linking for PC-type databases,ODBC linking and frequently asked questions. The linking processes are based on matching records, from a parent table (left table) to records in a child table (right table). The match is based on whether the value in the link field in the source record is identical to the value of the link field in the searched records. The linking process is a form of record selection.
